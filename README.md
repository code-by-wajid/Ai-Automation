# AI Automation 

A 100-day, self-directed execution roadmap to go from zero programming background to a client-ready AI Automation Consultant — built to run in parallel with the Tech7Academy "Master AI Agents" course.

This isn't a study log. It's an execution repo: every day ships something — a script, a workflow, an agent, a deliverable — committed to this repository the same day it's built.

## The Roadmap

- Day-by-day tasks across 5 phases (100 days total)
- A progress tracker that saves your checked-off days locally in the browser
- Skill matrix, readiness checklists, pricing packages, and a post-100-day growth plan

**Structure:**


| Phase                              | Days   | Focus                                               |
| ---------------------------------- | ------ | --------------------------------------------------- |
| 1 — Core Automation Infrastructure | 1–20   | JSON, HTTP, APIs, webhooks, n8n/Make                |
| 2 — AI-Assisted Vibecoding         | 21–40  | Cursor, AI debugging, local tools, niche commitment |
| 3 — Agentic AI                     | 41–60  | Tool-calling agents, memory, approval loops, RAG    |
| 4 — Enterprise Consulting          | 61–80  | Production reliability, pricing, discovery calls    |
| 5 — Go-To-Market                   | 81–100 | Outreach, sales, proposals, first paid client       |




## Repo Conventions

- **One folder per day** where a day produces its own files: `day-01-json-basics/`, `day-17-capstone-core/`, etc. Not every day needs a folder — some days are pure documentation or review.
- `api-cheatsheets/` — a growing personal reference library, one file per API integrated (started Day 6).
- `local-tools/` — standalone scripts/CLI tools built during Phase 2 (started Day 24).
- **Commit daily.** Every day's task list ends with an exact git commit message — use it as written for a clean, readable history.
- **Never commit** `.env` **or any file containing an API key.** See `.gitignore`.
